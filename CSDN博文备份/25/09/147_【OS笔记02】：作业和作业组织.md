### 作业和作业组织

在操作系统中， **“作业”** 是一个非常基础且重要的概念，它代表了用户交给计算机系统的一个独立的任务。

#### 一般编程过程

要理解什么是作业，我们可以先看一个熟悉的例子：

一个完整的编程过程。这个过程从最初的想法到最终程序输出结果，构成了一个完整的工作流程，这就可以被看作是一个“作业”。

这个过程大致包括以下步骤：
`概念` -> `需求分析` -> `功能设计` -> `结构设计` -> `详细设计` -> `编辑输入` -> `编译` -> ( `调试` ) -> `链接` -> `执行` -> `输出` -> `建档` 

#### 作业的定义

-  **作业 (Job)** ：用户要求计算机所做的有关该次业务处理的 **全部工作** 称为一个作业。

  - 例如，上面整个“编程过程”就是一个作业。另一个例子是“进行一次科学计算”，这可能包括读取数据、运行计算程序、输出结果三个步骤。

-  **作业步 (Job Step)** ：一个作业由不同的、顺序相连的 **作业步** 组成。

  -  **作业步** 是在一个作业的处理过程中，计算机所做的相对独立的、具体的工作。

  - 在编程这个作业中，“编译”是一个作业步，“链接”是另一个作业步，“执行”又是下一个作业步。这几个作业步按顺序完成后，整个作业才算完成。
